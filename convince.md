# 开发规范

## 代码规范

- 代码和注释使用英文 ，但是本项目的数据有中文，不强行规定不使用英文，但是中文优先。

## 代码风格

- Python: PEP 8
- 类型注解: 必须
- 单元测试覆盖: >= 80%
- Python 代码行末空行，此标准属于PEP 8，但是问题经常出现，单独列出
- Python 代码行长度100字符以内

### 禁止事项

- ❌ 硬编码 SQL 字符串拼接（必须参数化）
- ❌ 硬编码数据库/知识库路径（应可配置）
- ❌ 直接读取整个知识库文件（必须索引/检索）
- ❌ 泄露敏感字段（如 manager_id 映射关系需权限控制）
- ❌ 未经核实的数据推断

## 安全要求

- SQL 参数化查询，防注入
- 输入验证和过滤
- 数据源路径可配置
- 不暴露内部错误信息给用户
