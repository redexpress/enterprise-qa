# 企业智能问答助手 - 软件需求规格说明书

## 项目概述

### 项目名称
Enterprise QA System（企业智能问答助手）

### 项目目标
开发一个智能问答系统，能同时查询结构化数据（SQLite）和非结构化知识（Markdown文档），通过 Claude Code 提供服务。

## 功能需求

### 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 意图分类 | 判断问题类型：DB查询 / KB查询 / 混合查询 | 必须 |
| SQL执行 | 参数化查询，防SQL注入 | 必须 |
| 知识检索 | 文档索引与关键词匹配 | 必须 |
| 结果融合 | 多源信息综合，生成自然语言答案 | 必须 |
| 来源标注 | 标注答案来源（表名/文档名） | 必须 |

### 查询类型

| 类型 | 示例 | 数据源 |
|------|------|--------|
| 纯DB查询 | "张三的邮箱是什么？" | SQLite |
| 纯KB查询 | "年假怎么算？" | Markdown |
| 混合查询 | "王五符合晋升条件吗？" | SQLite + Markdown |
| 跨表查询 | "研发部有哪些在研项目？" | 多表JOIN |
| 时间范围 | "张三上个月迟到几次？" | DB + 日期计算 |

## 数据源

- 数据库（SQLite）
- 知识库（Markdown）

## 技术栈和工具

后端: Python 3.10+, SQLite
工具：Claude Code，安装测试skill
大模型：配合 Claude Code, 使用 MinMax M2.7 测试。
